## 思考题回答

### 在高维场景下，哪个 API 崩溃了或极其缓慢？为什么？

在高维场景下，**自制梯度下降求解器**表现极其缓慢，耗时达到了45.0564秒，远高于其他求解器。这是因为我们的自制梯度下降实现使用了全批量梯度下降（Batch Gradient Descent），每次迭代都需要计算所有样本的梯度，时间复杂度为O(N×P)。当特征维度P很高时（如2000个特征），每次迭代的计算量都会很大，导致整体训练时间显著增加。

相比之下，自制解析解、statsmodels.OLS和sklearn.LinearRegression虽然也需要处理高维数据，但它们使用的是更高效的矩阵运算库和算法，因此表现更好。

### 为什么 `SGDRegressor` 能在极短时间内完成任务？

`SGDRegressor`（随机梯度下降）能在极短时间内完成任务的原因有以下几点：

1. **随机采样**：SGD每次迭代只使用一个或一小批样本计算梯度，而不是所有样本，大大减少了每次迭代的计算量
2. **迭代优化**：SGD使用迭代方法逼近最优解，不需要计算和存储大型协方差矩阵
3. **高效实现**：sklearn的SGDRegressor实现了优化的算法，利用了向量化操作和其他性能优化技术
4. **时间复杂度**：SGD的时间复杂度为O(k×N×P)，其中k是迭代次数，通常远小于P，因此在高维场景下表现优异

从实验结果可以看出，SGDRegressor在高维数据上的耗时仅为0.3819秒，甚至比自制解析解还要快，充分展示了随机梯度下降在处理高维数据时的优势。
