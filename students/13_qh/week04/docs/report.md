## 实验配置
- 样本数量 N = 10000
- 低维特征 P = 10
- 高维特征 P = 2000
- 实验内容：对比 5 种线性回归求解器的耗时与精度

## 二、思考题回答
### 1. 高维场景下哪个 API 极其缓慢？
**Statsmodels.OLS** 在高维场景下速度极慢，耗时高达 16.38s，远高于其他求解器。

### 2. 为什么 Statsmodels.OLS 缓慢？
1. Statsmodels 专注于**统计推断**，会额外计算大量统计指标：p值、标准误、置信区间、协方差矩阵等。
2. 高维下矩阵运算量大，额外统计计算大幅增加耗时。
3. 未做工业级数值优化，更适合小数据、低维统计分析，不适合高维场景。

### 3. 为什么 SGDRegressor 能极快完成任务？
1. **SGD 是迭代算法，不需要计算大型矩阵的逆/分解**，时间复杂度远低于解析解。
2. 解析解复杂度 O(P³)，高维下计算量爆炸；SGD 复杂度 O(epochs×N×P)，高维更高效。
3. Sklearn 底层使用 C 语言优化，迭代速度极快。
4. 高维场景下，**迭代法天然优于矩阵解析法**。

### 4. SGDRegressor MSE 爆炸原因
未进行**特征标准化**，高维数据梯度值差异巨大，导致权重发散、预测值异常。
工业使用 SGD 必须先做 StandardScaler 标准化。
