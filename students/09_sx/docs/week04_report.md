## 五、思考题

### 5.1 高维场景下，哪个 API 崩溃了或极其缓慢？

**statsmodels.OLS** 最慢（5.9220 秒），手写梯度下降也很慢（5.4063 秒）。

**原因**：
- statsmodels 需要计算完整的统计推断，解析解需要求 (X^T X) 的逆矩阵
- 复杂度为 O(P³)，当 P=2000 时计算量巨大
- 手写梯度下降未做优化，收敛慢

### 5.2 为什么 SGDRegressor 能在极短时间内完成任务？

| 因素 | 说明 |
|------|------|
| 避免矩阵求逆 | 不需要计算 (X^T X)⁻¹ |
| 内存友好 | 内存需求 O(P) 而非 O(P²) |
| 凸优化 | 线性回归损失函数是凸函数，保证收敛 |
| 工程优化 | 内置特征缩放、自适应学习率 |
| 精度换速度 | 允许一定近似误差换取速度提升 |

## 六、五种方法比较

| 方法 | 低维耗时 | 高维耗时 | 高维MSE | 适用场景 |
|------|----------|----------|---------|----------|
| 手写解析解 | 0.0018s | 0.6289s | 0.0141 | 低维、教学 |
| 手写梯度下降 | 0.0416s | 5.4063s | 0.4267 | 教学 |
| statsmodels | 0.0096s | 5.9220s | 0.0141 | 统计推断 |
| sklearn LinearRegression | 0.0066s | 2.4764s | 0.0141 | 一般场景 |
| sklearn SGDRegressor | 0.0046s | 0.5007s | 0.0224 | 高维大数据 |

## 七、结论
1. 低维场景下，五种方法精度一致，解析解最快
2. 高维场景下，SGDRegressor 最快，statsmodels 最慢
3. 手写梯度下降在高维时精度严重下降（MSE 0.4267）
4. 高维大数据推荐使用 SGDRegressor
